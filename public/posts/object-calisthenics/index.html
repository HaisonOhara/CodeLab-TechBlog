Object Calisthenics: Don’t Use Else#

Olá pessoal! Hoje vou falar sobre uma das regras mais interessantes do Object Calisthenics: a regra “Don’t Use Else” (Não Use Else).

Object Calisthenics é um conjunto de 9 regras de exercícios de programação que nos ajudam a escrever código mais limpo, mais legível e mais orientado a objetos. A regra “Don’t Use Else” propõe que evitemos o uso da palavra-chave else em nosso código, o que nos força a pensar de maneira diferente sobre o fluxo de controle.

Por que evitar o else?#

Quando eliminamos o else do nosso código, somos obrigados a:

  • Usar retornos antecipados (early returns)
  • Aplicar polimorfismo
  • Utilizar padrões de design mais sofisticados
  • Escrever código mais linear e fácil de seguir
